Brèves du jour

10 juin — Visa

Visa ajoute un annuaire d'agents au commerce

Visa Intelligent Commerce inclut désormais Agent Score et Agentic Directory. Le premier évalue la lisibilité d'un site pour agents ; le second vérifie agents et marchands afin que chaque côté sache avec qui il transige. Source : Visa.

10 juin — OpenAI

OpenAI et Visa branchent le paiement sur l'agent

Le partenariat prévoit des paiements Visa dans des expériences OpenAI, avec permissions définies par l'utilisateur : plafonds, catégories de marchands, approbations, credentials tokenisés et monitoring de fraude. Source : Visa.

10 juin — Mastercard

Mastercard prépare les paiements machine-à-machine

Agent Pay for Machines vise les transactions automatisées entre agents et logiciels. CoinDesk cite plus de 30 participants, dont Coinbase, Stripe, Adyen, Checkout.com, Cloudflare et OKX. Source : CoinDesk.

5 juin — sécurité

Microsoft documente une fuite de secrets dans un workflow agentique

Claude Code GitHub Action pouvait lire /proc/self/environ via son outil Read, hors sandbox Bash, et exposer ANTHROPIC_API_KEY. Anthropic a corrigé en version 2.1.128 le 5 mai. Source : Microsoft Security.

12 juin — OpenClaw

OpenClaw resserre ses frontières de sécurité

La version 2026.6.6 annonce des limites plus strictes sur transcripts, sandbox binds, héritage d'environnement hôte, MCP stdio, accès HTTP Codex et outils loopback. Le dépôt affiche 379 000 étoiles. Source : GitHub.

À la une · Paiements & identité

Les réseaux de paiement veulent reconnaître l'agent avant qu'il paie.

Visa et Mastercard ne parlent plus seulement de commerce agentique comme d'une interface. Elles construisent les rails : annuaires d'agents, tokens, limites de dépense, règlement garanti. La question centrale se déplace : moins « que peut acheter l'agent ? » que « comment sait-on que c'est le bon agent ? »

Par La rédaction · synthèse de sources · 7 min de lecture

La semaine a donné une forme bancaire au mot « agent ». Le 10 juin, au Visa Payments Forum, Visa a annoncé un paquet d'outils pour le commerce agentique : Agent Score, pour mesurer si un site marchand peut être compris et utilisé par des agents ; Agentic Directory, pour vérifier agents et marchands ; et une collaboration avec OpenAI pour permettre des paiements Visa dans des expériences OpenAI, sous permissions explicites — limites de dépense, catégories de marchands, validations requises, credentials tokenisés et surveillance de fraude en temps réel. Le même jour, Mastercard poussait plus loin son Agent Pay avec Agent Pay for Machines, décrit par CoinDesk comme un service pour paiements automatisés entre agents et logiciels, à travers cartes, comptes bancaires et stablecoins, avec authentification, règles de dépense et règlement garanti ; plus de 30 entreprises sont citées dans l'initiative, et les permissions doivent d'abord être enregistrées sur Polygon, Solana et Base. Ce n'est pas encore l'économie autonome ; c'est son guichet d'identité. Le contraste vient de la sécurité : Microsoft a détaillé le 5 juin comment un workflow Claude Code GitHub Action pouvait exposer des secrets CI/CD via un outil de lecture non isolé, corrigé côté Anthropic en mai. Les agents apprennent donc à payer au moment précis où l'industrie apprend à les enfermer.

▦ Paiements · Visa

L'agent entre dans le paiement par l'identité vérifiée

Visa ne vend pas seulement une carte pour agents : l'entreprise propose une couche de confiance. Agent Score note la capacité d'un site à être parcouru par des agents ; Agentic Directory vérifie les agents et les marchands ; le partenariat OpenAI ajoute credentials tokenisés, plafonds, catégories et approbations. C'est le vocabulaire classique du paiement — autorisation, fraude, litige — appliqué à un acteur non humain. Le détail le plus parlant est un prototype en ligne de commande où un agent paie des services numériques avec des credentials Visa tokenisés. Visa, 10 juin.

▦ Machines · Mastercard

Mastercard regarde le paiement comme conversation entre logiciels

Agent Pay existait déjà comme programme de paiements agentiques avec Microsoft et les tokens agentiques Mastercard. La nouveauté de juin est plus mécanique : Agent Pay for Machines vise des transactions automatisées entre agents et logiciels, y compris cartes, comptes et stablecoins. CoinDesk cite plus de 30 entreprises et un premier enregistrement des permissions sur Polygon, Solana et Base. Le signal faible : Mastercard observe déjà des déclins autour de HTTP 402, faute de moyen de paiement disponible pour l'automatisation. Mastercard, CoinDesk.

▦ Travail · Codex

Codex quitte le seul code et devient outil de bureau

OpenAI affirme que plus de 5 millions de personnes utilisent Codex chaque semaine, et que les non-développeurs représentent environ 20 % de l'usage, avec une croissance trois fois plus rapide que celle des développeurs. Les six plugins annoncés couvrent data analytics, production créative, vente, design produit, investissement actions et banque d'affaires ; ils agrègent 62 applications et 110 skills. Avec Sites, Codex peut aussi produire des mini-applications partagées par URL dans l'espace de travail. L'agent de code devient agent de bureau. OpenAI, 2 juin.

Produit · 4 min
▦ Sécurité · CI/CD

Un outil de lecture suffit à déplacer la frontière

Le cas Microsoft n'est pas qu'une vulnérabilité Claude Code : c'est une leçon d'architecture. Dans un workflow GitHub, les issues, PR et commentaires sont des entrées non fiables ; l'agent lit, raisonne, puis invoque des outils dans un runner qui peut contenir secrets, tokens et droits d'écriture. Ici, le sandbox Bash existait, mais l'outil Read n'y passait pas. Microsoft en tire une règle pratique : ne jamais donner simultanément à un agent entrée non fiable, secrets sensibles et capacité de changer l'état ou de communiquer dehors. Microsoft Security, 5 juin.

Le commerce agentique commence par une question d'identité.

Les annonces de Visa et Mastercard font entrer l'agent dans les rails du paiement. Leur point commun : avant de lui donner une carte, il faut savoir qui il est, ce qu'il peut faire et qui répond de ses actes.

On croyait que la question du commerce agentique serait celle de l'interface : parler à ChatGPT, laisser un bot comparer les prix, demander à un assistant de réserver un vol. Les annonces de juin déplacent le sujet. Visa et Mastercard parlent moins de conversation que d'authentification, de token, de permissions et de règlement. C'est un signe de maturité : le paiement n'accepte pas longtemps les métaphores.

Chez Visa, le vocabulaire est explicite. Agent Score mesure si un site marchand est lisible par des agents ; Agentic Directory veut vérifier agents et marchands ; le partenariat OpenAI encadre les paiements par plafonds, catégories, validations requises, credentials tokenisés et monitoring de fraude. Chez Mastercard, Agent Pay for Machines formule la même obsession côté machine-à-machine : authentifier l'agent, appliquer des limites, garantir le règlement et inscrire les permissions dans une infrastructure vérifiable.

Le contrechamp de sécurité explique pourquoi cette grammaire arrive maintenant. Microsoft a montré qu'un workflow CI/CD agentique peut devenir un périmètre poreux dès qu'il combine entrées non fiables, secrets et outils de lecture ou de communication. Anthropic répond par sandboxing fichier + réseau ; Arkose chiffre l'attente d'incident à 97 % des responsables interrogés ; Cyera recense des centaines de dommages attribués à des agents. Le paiement agentique n'est donc pas une extension du bouton « acheter ». C'est une nouvelle identité non humaine dans l'économie.

◆ Édito · La rédaction

Avant le portefeuille agentique, il faut l'état civil de l'agent.

La tentation est de traiter le paiement agentique comme une fonction : ajouter une carte, définir un plafond, laisser l'assistant commander. C'est trop petit. Ce que Visa et Mastercard dessinent cette semaine ressemble moins à un bouton d'achat qu'à un état civil pour logiciels qui agissent.

Un agent qui paie n'est pas seulement un logiciel avec une permission. Il devient une partie à la transaction : il choisit, déclenche, justifie, parfois se trompe. Le marchand doit savoir s'il parle à un agent légitime ; l'émetteur doit savoir quelle instruction humaine l'autorise ; l'utilisateur doit pouvoir annuler, contester, limiter. Sans ces attributs, le paiement n'est qu'une délégation opaque.

Notre thèse est simple : l'économie agentique ne commencera pas quand un bot achètera une paire de chaussures. Elle commencera quand on pourra relire, après coup, qui a demandé, quel agent a exécuté, sous quelle règle, avec quel recours. L'agent n'a pas encore besoin de personnalité juridique. Il a déjà besoin de papiers.
